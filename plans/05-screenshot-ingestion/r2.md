# Screenshot Ingestion — R2 Client

Bytes never flow through the API process. The desktop client `PUT`s straight to R2 with a presigned URL; admins `GET` thumbnails the same way. The API only signs URLs and writes metadata. This document specifies how the API talks to R2.

## Provider choice

**Cloudflare R2 with the AWS S3 SDK v3.** R2 implements an S3-compatible API; using the standard SDK keeps us on a battle-tested codepath, lets us stub it for tests, and lets us swap to AWS S3 / MinIO / DigitalOcean Spaces without touching the calling code. We do **not** use a vendor-specific R2 SDK.

The two relevant packages:

```
@aws-sdk/client-s3              — the S3 client (PutObject / GetObject / DeleteObject)
@aws-sdk/s3-request-presigner   — wraps the client to generate presigned URLs
```

Install via `pnpm add @aws-sdk/client-s3 @aws-sdk/s3-request-presigner`.

## Env vars (all optional)

Already declared in [`apps/api/src/config/env.ts`](../../apps/api/src/config/env.ts) from Plan 00:

```
R2_ACCOUNT_ID         — your Cloudflare account id (used to build the endpoint URL)
R2_ACCESS_KEY_ID      — R2 access key (issued from Cloudflare dashboard)
R2_SECRET_ACCESS_KEY  — paired secret
R2_BUCKET             — bucket name
```

Plus one new env var to add:

```
R2_PUBLIC_URL_BASE    — optional; if R2 has a Custom Domain or Public Bucket URL,
                        we use it for thumbnail GETs to skip presigning entirely
                        on public-facing reads (we don't enable this in v1).
```

For v1 we always presign GETs. The base URL is reserved for a later optimization.

## Endpoint URL format

R2's S3 endpoint is:

```
https://<R2_ACCOUNT_ID>.r2.cloudflarestorage.com
```

Region is always `auto` for R2 (the S3 SDK requires _something_).

## Lazy client construction

Like the mail provider in Plan 03, the R2 client is **not** created at module load. It's lazily constructed on first call, so endpoints that don't need R2 don't crash on missing env vars.

```ts
// apps/api/src/lib/r2.ts
import { S3Client } from '@aws-sdk/client-s3';
import {
  GetObjectCommand,
  PutObjectCommand,
  DeleteObjectCommand,
  HeadObjectCommand,
} from '@aws-sdk/client-s3';
import { getSignedUrl } from '@aws-sdk/s3-request-presigner';

import { config } from '../config/env.js';
import { AppError } from './errors.js';
import { logger } from './logger.js';

interface R2 {
  client: S3Client;
  bucket: string;
}

let r2: R2 | null = null;

const init = (): R2 | null => {
  const acct = config.R2_ACCOUNT_ID;
  const key = config.R2_ACCESS_KEY_ID;
  const secret = config.R2_SECRET_ACCESS_KEY;
  const bucket = config.R2_BUCKET;
  if (!acct || !key || !secret || !bucket) {
    logger.warn('R2 not configured; ingestion endpoints will return 503');
    return null;
  }
  const client = new S3Client({
    region: 'auto',
    endpoint: `https://${acct}.r2.cloudflarestorage.com`,
    credentials: { accessKeyId: key, secretAccessKey: secret },
  });
  return { client, bucket };
};

const provider = (): R2 => {
  if (!r2) r2 = init();
  if (!r2) throw new AppError('r2_unavailable', 503, 'object storage not configured');
  return r2;
};

/** Test seam — replace the provider with a stub for integration tests. */
export const __setR2Provider = (next: R2 | null): void => {
  r2 = next;
};
```

## Public functions

The module exports three signed-URL helpers and one delete helper. Everything outside `r2.ts` calls these — never the SDK directly.

```ts
const PUT_TTL_S = 5 * 60; // 5 min — desktop has time to upload
const THUMB_TTL_S = 10 * 60; // 10 min — admin grids cache for the page lifetime
const FULL_TTL_S = 5 * 60; // 5 min — short-lived; refresh per click

export interface PresignedPut {
  putUrl: string;
  expiresAt: Date;
  key: string;
}

export const presignPut = async (
  key: string,
  contentType: string,
  maxSizeBytes: number,
): Promise<PresignedPut> => {
  const { client, bucket } = provider();
  const cmd = new PutObjectCommand({
    Bucket: bucket,
    Key: key,
    ContentType: contentType,
    ContentLength: maxSizeBytes, // upper bound — uploads larger than this are rejected by R2
  });
  const putUrl = await getSignedUrl(client, cmd, { expiresIn: PUT_TTL_S });
  return { putUrl, key, expiresAt: new Date(Date.now() + PUT_TTL_S * 1000) };
};

export const presignGetThumbnail = async (
  key: string,
): Promise<{ url: string; expiresAt: Date }> => {
  const { client, bucket } = provider();
  const cmd = new GetObjectCommand({ Bucket: bucket, Key: key });
  const url = await getSignedUrl(client, cmd, { expiresIn: THUMB_TTL_S });
  return { url, expiresAt: new Date(Date.now() + THUMB_TTL_S * 1000) };
};

export const presignGetFull = async (key: string): Promise<{ url: string; expiresAt: Date }> => {
  const { client, bucket } = provider();
  const cmd = new GetObjectCommand({ Bucket: bucket, Key: key });
  const url = await getSignedUrl(client, cmd, { expiresIn: FULL_TTL_S });
  return { url, expiresAt: new Date(Date.now() + FULL_TTL_S * 1000) };
};

export const deleteObject = async (key: string): Promise<void> => {
  const { client, bucket } = provider();
  await client.send(new DeleteObjectCommand({ Bucket: bucket, Key: key }));
};

export const headObject = async (key: string): Promise<{ size: number } | null> => {
  const { client, bucket } = provider();
  try {
    const out = await client.send(new HeadObjectCommand({ Bucket: bucket, Key: key }));
    return { size: out.ContentLength ?? 0 };
  } catch {
    return null;
  }
};

/** Used by the worker — fetches an object as a Buffer for sharp processing. */
export const getObjectBytes = async (key: string): Promise<Buffer> => {
  const { client, bucket } = provider();
  const out = await client.send(new GetObjectCommand({ Bucket: bucket, Key: key }));
  if (!out.Body) throw new Error(`empty body for ${key}`);
  // The SDK returns a stream in Node; collect to a buffer.
  const chunks: Buffer[] = [];
  for await (const chunk of out.Body as AsyncIterable<Uint8Array>) {
    chunks.push(Buffer.from(chunk));
  }
  return Buffer.concat(chunks);
};

export const putObjectBytes = async (
  key: string,
  bytes: Buffer,
  contentType: string,
): Promise<void> => {
  const { client, bucket } = provider();
  await client.send(
    new PutObjectCommand({
      Bucket: bucket,
      Key: key,
      Body: bytes,
      ContentType: contentType,
    }),
  );
};
```

## Key scheme

Object keys are computed in the screenshots service, never in `r2.ts`. The scheme partitions by org and date so retention sweeps can target prefixes:

```
orgs/{orgId}/users/{userId}/{yyyy}/{mm}/{dd}/{screenshotId}.jpg
orgs/{orgId}/users/{userId}/{yyyy}/{mm}/{dd}/{screenshotId}-thumb.webp
orgs/{orgId}/users/{userId}/{yyyy}/{mm}/{dd}/{screenshotId}-blur.jpg
```

`{yyyy}/{mm}/{dd}` derives from `capturedAt` in UTC (per the convention from [docs/04-data-model.md:7](../../docs/04-data-model.md#L7)).

A small helper:

```ts
// apps/api/src/lib/screenshot-keys.ts
export interface KeyParts {
  orgId: string;
  userId: string;
  capturedAt: Date;
  screenshotId: string;
}

const datePath = (d: Date): string => {
  const y = d.getUTCFullYear();
  const m = String(d.getUTCMonth() + 1).padStart(2, '0');
  const day = String(d.getUTCDate()).padStart(2, '0');
  return `${y}/${m}/${day}`;
};

const extFromContentType = (ct: string): string => {
  if (ct === 'image/png') return 'png';
  if (ct === 'image/webp') return 'webp';
  return 'jpg'; // image/jpeg default
};

export const originalKey = (p: KeyParts, contentType: string): string =>
  `orgs/${p.orgId}/users/${p.userId}/${datePath(p.capturedAt)}/${p.screenshotId}.${extFromContentType(contentType)}`;

export const thumbnailKey = (p: KeyParts): string =>
  `orgs/${p.orgId}/users/${p.userId}/${datePath(p.capturedAt)}/${p.screenshotId}-thumb.webp`;

export const blurredKey = (p: KeyParts, contentType: string): string =>
  `orgs/${p.orgId}/users/${p.userId}/${datePath(p.capturedAt)}/${p.screenshotId}-blur.${extFromContentType(contentType)}`;
```

## Local dev (without paying for R2)

For dev work without an R2 bucket, two options:

1. **MinIO running locally** with the same env-var shape pointing at `http://localhost:9000`. Works because the SDK is generic. Useful when you want to debug the full bytes flow.
2. **Skip R2 entirely.** Endpoints needing it return `503 r2_unavailable`. The other endpoints (auth, orgs, projects, time entries with no screenshots) keep working. The integration tests use a stub provider; the unit tests for `r2.ts` itself mock `@aws-sdk/client-s3` directly.

Most of this plan's tests use the stub. End-to-end smoke testing (per the README's done-when criteria) requires a real R2 bucket _or_ MinIO.

## Why we don't open the bucket publicly

Cloudflare R2 supports [Public Buckets](https://developers.cloudflare.com/r2/buckets/public-buckets/) and Custom Domains that bypass presigning. We don't use them in v1 because:

- Presigned URLs let us **revoke access in seconds** (just stop signing). With public access, a leaked URL works forever.
- Per-org access control is impossible with public buckets — anyone who knows a key can read.
- The cost of presigning is one round-trip on each list response and one on each modal open. Acceptable for our scale.

If we ever onboard an external customer with public-portfolio screenshots, we'll revisit per project.

## What this module does NOT do

- **No retry logic in `r2.ts`.** The SDK handles transient errors. The worker's outer retry policy handles harder failures.
- **No client-side encryption.** Per [docs/07-screenshot-pipeline.md:117](../../docs/07-screenshot-pipeline.md#L117), out of scope for v1.
- **No multi-part upload.** Screenshots are bounded at 8 MB; single PUT is plenty.
- **No bucket-creation code.** Bucket is created via the Cloudflare dashboard once, manually. We don't want the API runtime to have CreateBucket permissions.
